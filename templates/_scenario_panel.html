Anatomy of a Proof Reading guide for non-mathematicians
axiom

An undeniable medical fact the kernel is allowed to start from. Each axiom in this file is either a published clinical guideline (e.g. AHA_ACC_HTN_8_1_5) or one of the patient's established findings drawn from the chart (e.g. obs_severe_dehydration). Lean accepts these as ground truth and never tries to prove them.

theorem / absurd

A claim the kernel must verify before it is accepted — unlike an axiom, a theorem is only true if it is mechanically proven from the surrounding axioms. The audit's final theorem is named absurd: its goal is False, so successfully proving it means the encoded treatment plan is a logical impossibility for this patient. That is the contradiction the audit exists to surface.

Prop

Short for Proposition: a statement Lean can evaluate as strictly true or false. When the file declares Indicated : Patient → Treatment → Prop, it is saying "whether a treatment is indicated for a patient is a true-or-false claim" — exactly the kind of statement the kernel knows how to reason about.
